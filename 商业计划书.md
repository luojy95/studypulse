## 一、项目概述

### 1.1 一句话定位

**学脉 = 作业帮的精准度 + Duolingo的上瘾机制 + 小红书的社区粘性**

### 1.2 核心问题

中国 2,400 万高中生中：
- **72%** 的学生不知道自己具体哪个知识点薄弱，只能盲目刷题
- **85%** 的家长无法实时了解孩子真实学习状况，只能等考试成绩
- **90%** 的老师无法对 45+ 名学生进行一对一精准辅导
- 现有产品（作业帮、猿辅导）重「搜题」轻「追踪」，用完即走，零黏性

### 1.3 我们的解法

构建 **「做题 → AI分析 → 精准推荐 → 社区激励 → 持续成长」** 的完整闭环：

```
┌─────────────────────────────────────────────────────────────┐
│                    学脉产品闭环                               │
│                                                              │
│    做题 ──→ AI实时分析 ──→ 精准推荐 ──→ 社区分享             │
│     ↑                                        │              │
│     └────────── 持续学习动力 ←────────────────┘              │
│                                                              │
│    数据积累 → AI更准 → 体验更好 → 用户更多 → 数据更多        │
│                     （数据飞轮）                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 产品名称释义

**学脉（StudyPulse）**
- 「学」= 学习
- 「脉」= 脉搏（实时检测学习心跳）+ 脉络（知识图谱的脉络）+ 人脉（社区的连接）
- 英文 Pulse = 脉搏，呼应产品核心：实时追踪每一次做题的"心跳"

### 4.2 知识追踪模型 (Knowledge Tracing)

**混合模型架构：DKT + BKT + Attention**

```python
class PulseTracker(nn.Module):
    """
    学脉知识追踪模型
    输入：学生做题序列 [(知识点ID, 正误), ...]
    输出：各知识点掌握概率 [0, 1]
    """
    def __init__(self, n_kp=126, d_embed=128, d_hidden=256):
        super().__init__()
        self.kp_embed = nn.Embedding(n_kp, d_embed)       # 知识点嵌入
        self.ans_embed = nn.Embedding(2, d_embed)          # 正误嵌入
        self.lstm = nn.LSTM(d_embed*2, d_hidden, batch_first=True, num_layers=2)
        self.attention = nn.MultiheadAttention(d_hidden, num_heads=8)
        self.out = nn.Sequential(
            nn.Linear(d_hidden, d_hidden),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(d_hidden, n_kp),
            nn.Sigmoid()
        )

    def forward(self, kp_seq, ans_seq):
        # kp_seq: (batch, seq_len) 知识点ID序列
        # ans_seq: (batch, seq_len) 正误序列 0/1
        k = self.kp_embed(kp_seq)            # (B, L, D)
        a = self.ans_embed(ans_seq)           # (B, L, D)
        x = torch.cat([k, a], dim=-1)        # (B, L, 2D)
        h, _ = self.lstm(x)                  # (B, L, H)
        h_attn, _ = self.attention(h, h, h)  # Self-Attention
        mastery = self.out(h_attn)            # (B, L, 126)
        return mastery  # 每个时间步对所有知识点的预测掌握度
```

**模型特点：**
- **LSTM** 捕捉做题顺序依赖（昨天做错的知识点影响今天的判断）
- **Self-Attention** 捕捉远距离知识关联（力学薄弱可能影响天体运动）
- **2层LSTM + Dropout** 防止过拟合
- **126维输出** 同时预测所有知识点的掌握度

**训练数据规模（目标）：**
- 10万+学生的做题序列
- 平均每人500+条做题记录
- 总计5000万+条序列数据

### 4.3 推荐引擎

**三路召回 + LTR 排序架构：**

```
用户画像 ─────┬──── 协同过滤召回 ────┐
              │     (相似学生做的题)    │
              ├──── 内容召回 ─────────┤──→ 候选池 ──→ LTR排序 ──→ 重排 ──→ Top-K
              │     (同知识点/同题型)    │
              └──── 策略召回 ─────────┘
                    (薄弱优先/遗忘优先)

LTR 排序特征：
├── 学生特征：当前mastery、历史正确率、学习时段
├── 题目特征：难度、题型、知识点、来源
├── 交叉特征：mastery-difficulty差值、知识点前置依赖完成度
└── 上下文特征：当前session做题数、连续正确/错误数
```

**ZPD（最近发展区）理论实现：**

```
推荐难度 = 学生当前掌握度 + Δ

其中 Δ 的动态调整：
- 连续3题正确 → Δ 增大（增加挑战）
- 连续2题错误 → Δ 减小（降低难度，恢复信心）
- 默认 Δ = 0.1~0.15（略高于当前水平）
```

### 4.5 遗忘曲线 & 间隔重复

**艾宾浩斯遗忘曲线实现：**

```
记忆保持率 R(t) = e^(-t/S)

其中：
- t = 距上次练习的天数
- S = 记忆强度（由做对次数和间隔决定）

当 R(t) < 0.6 时 → 触发复习提醒
复习间隔梯度：1天 → 3天 → 7天 → 14天 → 30天
```

**在产品中的体现：**
- Pop-up提醒："「光学」已7天未练，建议复习"

## 五、社区运营深度方案（学脉圈）

### 5.1 社区定位

**学脉圈 = 高中生的学习版小红书**

核心差异化：
- 小红书是「消费内容」，学脉圈是「生产学习成果」
- 发帖动力来源于学习过程中的自然产出（解题笔记、错题分析、打卡数据）
- 不靠猎奇/颜值/生活方式驱动，靠「知识获得感」和「同伴压力」驱动

### 5.2 内容生态

**五大内容类型：**

| 类型 | 占比目标 | 内容来源 | 激励机制 |
|------|---------|---------|---------|
| 📝 解题笔记 | 35% | 学生主动分享解题思路 | 被收藏=高学脉值 |
| 💡 错题分析 | 20% | AI辅助生成（一键分享错题） | 帮助他人=勋章 |
| 🏆 打卡成就 | 25% | 系统自动生成（日历/成绩） | 连续打卡奖励 |
| ❓ 求助帖 | 10% | 学生主动提问 | 被采纳=额外XP |
| ✨ 经验分享 | 10% | 提分故事/方法论 | 高赞=推荐加权 |

### 5.3 社区架构

```
学脉圈社区分层
├── 公域（全国可见）
│   ├── 推荐流（AI排序，混合热门+个性化）
│   ├── 话题广场（#向心力专题 #高考倒计时）
│   └── 排行榜（全国/省/市）
│
├── 学校域（本校可见）
│   ├── 校内动态
│   ├── 校内排名
│   └── 老师发布的内容
│
├── 班级域（本班可见）
│   ├── 班级群聊
│   ├── 班级排名
│   ├── 老师布置的作业
│   └── 班级每日挑战完成情况
│
└── 家长域（家长专属）
    ├── 家长交流
    ├── 教育经验
    └── 学情讨论
```

### 5.4 社区增长飞轮

```
新用户做题 → 产生学习数据 → AI生成个性化内容（"恭喜！向心力掌握度提升15%"）
     ↓
一键分享到学脉圈 → 同学看到 → "我也想提升" → 下载/使用学脉
     ↓
更多用户 → 更多内容 → 社区更活跃 → 更多人留下来
     ↓
社区黏性 → 留存率 ↑ → LTV ↑ → 付费转化 ↑
```

### 5.5 社区运营节奏

| 阶段 | 时间 | 策略 | 目标 |
|------|------|------|------|
| 冷启动 | M1-2 | 50个种子用户+教研团队生产内容 | 200帖/天 |
| 增长期 | M3-6 | 鼓励UGC（发帖=XP奖励） | 2000帖/天 |
| 爆发期 | M7-12 | KOL合作+话题运营+全校部署 | 1万帖/天 |
| 成熟期 | Y2+ | AI辅助内容审核+推荐算法优化 | 10万帖/天 |

### 5.6 社区安全

- AI内容审核（敏感词过滤 + 图片识别）
- 实名制（学校验证）
- 举报机制（24小时响应）
- 未成年人保护（夜间使用限制、防沉迷）
- 页面内HTML转义防XSS攻击
